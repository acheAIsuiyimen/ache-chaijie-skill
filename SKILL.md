# ache chaijie skill

## 目标

回答四个问题：账号如何定位；什么内容有效；哪些方法可复制；用户下一步应该做什么。

## 入口路由

按顺序判断，命中后立即执行：

1. **消息已有 1-5 个小红书账号链接**：直接分析，不问前置问题，不搜索候选账号。
2. **用户说已有账号，但未发链接**：只请对方提供主页链接；收到后直接分析。
3. **用户要找竞品/对标**：第一轮先询问是否已有对标账号；有则请其直接发送 1-5 个主页链接，没有才执行“对标发现”。
4. **意图不明**：只问：“A. 分析已有账号；B. 先找对标账号？”

直链模式的第一句：

> 收到链接。我会直接基于公开主页、可见笔记、互动数据和封面做拆解；未公开数据会标注，不会编造。

## 对标发现

只有用户需要找账号时才询问。不要把“电商、美妆、教育、职场、知识”等大类词当作完整定位，也不要在定位不清时开始搜索。

第一轮必须同时给出已有对标账号的直达入口，不得等用户回答定位后才询问：

> 你手上已经有想拆解的对标账号吗？有的话直接发 1-5 个小红书主页链接，我会跳过筛选并开始分析；没有的话，请告诉我：做/卖什么、由谁出镜或运营、希望用户看完做什么。比如“女装店主真人试穿，引导进店下单”。

根据回答动态生成最多 4 个选择题，补齐已缺失的信息；用户说过的不再问：

1. **具体品类/主题**：必须明确到可用于搜索的关键词，如通勤女装、家清用品、宠物食品、电商运营教学。
2. **账号类型/身份**：品牌或店铺 / 店主或创始人 / 买手或测评 / 运营知识 / 工厂供应链 / 直播电商 / 其他。
3. **目标用户与场景**：谁在什么情况下需要这些内容。
4. **主要内容形态**：真人讲解 / 产品演示测评 / 场景 UGC / 直播切片 / 图文清单 / 不露脸流程。
5. **核心目标**：店铺下单 / 直播间引流 / 留资或私域 / 品牌认知 / 咨询课程 / 涨粉起号。

搜索前先复述：

> 我理解你要做的是：{账号类型}，聚焦{具体品类}，面向{目标用户/场景}，主要用{内容形态}，目标是{核心目标}。如果没问题，我再按这个画像找对标账号。

必须等用户确认，或明确说“按默认理解先搜”，才能进入搜索。若品类、账号类型、目标用户中仍有两项不清楚，继续追问，不推荐账号。

根据回答推荐 5 个账号：2 个成熟标杆 + 3 个当前可对标。表格仅保留：账号与链接、公开体量/热度、定位、代表内容、适合学习点、不宜照搬点。

让用户确认 1-5 个账号后再做完整拆解。

## 能力路由

先检查当前可用能力，不假设特定插件、登录、Cookie 或本机路径存在。

- 已有账号链接：优先浏览器/网页读取；有社交连接器时可用其补充。
- 搜索对标：优先社交连接器或平台搜索；其次使用通用网页搜索。
- 开始搜索前先声明访问状态：`站内可访问`、`仅外部网页搜索`或`仅用户材料`。
- `站内已验证`：实际打开小红书搜索页、主页或笔记，并取得可点击的小红书链接。
- `外部搜索待验证`：仅从搜索引擎、文章或榜单发现；必须逐项标注“待站内确认”，不能写成已搜索小红书，也不能据此输出正式热度排名。
- 无法读取平台：提供三个选项：授权本地浏览器访问；用户提供账号链接/截图；接受仅供初筛的外部候选。不要默认跳到截图方案。
- 首次无法访问时不要立即进入兜底。先执行“访问恢复”：检查当前 App 的内置工具、Skills、连接器、浏览器控制和本地自动化能力；主动询问宿主是否存在可行方案，并尝试获准使用的内置路径。
- 只有完成访问恢复仍失败时，才简要说明具体限制，读取 [references/tools.md](references/tools.md)，同时提供外部连接器和截图提交方案。
- 图片不能本地化：保留原链接和文字拆解，不中止报告。

只在工具缺失或用户询问安装时，读取 [references/tools.md](references/tools.md) 并推荐可选工具。安装、读取 Cookie 或登录前必须获得用户授权。

候选账号必须包含可点击的小红书主页链接。没有主页链接、无法确认账号存在或只有二手文章提及的账号，不得列入“已确认对标”；严禁凭行业印象生成账号名、粉丝量、代表内容或热度。

## 访问恢复

收到“无法获取、无权限、当前不支持、不能访问”等结果时，按顺序执行，不把第一次拒绝当作最终结论：

1. 列出当前实际可用的浏览器、Skills、插件、MCP/连接器、Computer Use 或网页工具。
2. 询问宿主 App：“是否有能控制本地浏览器、读取当前标签页、调用已安装 Skill，或安装社交平台连接器的方案？”
3. 优先尝试内置路径：控制用户已打开的浏览器；让用户手动登录后读取公开页；调用现有社交/浏览器 Skill；从官方技能市场安装可用连接器。
4. 涉及安装、登录、Cookie、浏览器权限时，先说明动作和风险并取得用户授权；不索要密码，不绕过验证码或风控。
5. 内置方案均不可用后，再提供外部连接器、外部搜索待验证或截图材料三种路径。

需要向宿主 App 提问时，读取 [references/tools.md](references/tools.md) 中的“给宿主 App 的排障问法”，结合当前工具名称提问，不机械照抄全部示例。

## 数据采集

每个账号尽量获取：

- 账号名、链接、头像、简介、粉丝、获赞与收藏、置顶内容。
- 主页可见笔记的标题、链接、日期和可见互动数；选取 TOP 5-10 详细阅读。
- TOP 笔记的正文、点赞、收藏、评论、封面与评论区需求。

数据规则：

- 主页只有一个数字时，写“主页可见互动数”，不自行判定是点赞或收藏。
- 只在笔记详情明确区分时，分别写点赞、收藏、评论。
- 没有浏览量等分母时，不计算“收藏率”。
- 无法获取的字段标注“未公开/未获取”，不编造。
- 报告标注数据日期及“公开可见数据”或“用户提供数据”。
- 只有外部网页搜索时，不分析“实时粉丝、TOP 笔记、真实封面”；先取得站内证据或用户材料。

## 分析与个性化

提炼：一句话定位、受众、内容主线、账号演化阶段、爆款与低效内容差异、标题公式、封面公式、评论需求、可复制点和风险。

开始正式分析时必须读取 [references/xhs-knowledge-base.md](references/xhs-knowledge-base.md) 和 [references/xhs-analysis.md](references/xhs-analysis.md)，用前者建立判断基线，用后者执行封面、内容、爆款复用和账号演化审计。

- 有图像/视觉能力：实际查看 TOP 封面原图或完整截图，再描述封面文字、主体、构图和视觉钩子。
- 无图像/视觉能力：先执行访问恢复；仍不可用时请求 3-5 张完整封面，或明确标注“未做视觉审计”。严禁根据标题、文件名或笔记正文猜测封面内容。
- 只分析当前实际看见的页面、图片、文字、评论和公开数字。对内容作用的解释可标“合理推断”；不可见的后台指标和转化结果直接省略，不把经验规律写成官方算法。

- **已知用户定位**：明确写适合学什么、不适合学什么、如何迁移到用户的领域。
- **未知用户定位**：只做账号本身拆解和通用建议；不生成虚假的“匹配度/学习指数”。完成报告后，可选询问领域、受众、目标 3 项以追加个性化建议。

不要将大账号整体照搬给小账号；必须拆成当前可复制模块。

## 报告输出

核心报告按顺序包含：

1. 账号概览与核心判断。
2. 账号演化：早期、转折期、近期的可见变化；证据不足时省略。
3. 公开笔记热度榜及爆点/低效原因。
4. 内容主线、用户需求和标题/封面公式。
5. 爆款复用：哪些母题、标题句式和封面模板被重复，重复后的公开数据是否稳定。
6. 真实封面视觉审计：样本、逐张拆解、共性模式和可复制模板。
7. 可复制点与不宜照搬点。

只在用户已提供自身定位，或明确要求行动方案时，增加：迁移建议、5-10 个选题、第 1 篇内容和下一阶段行动。4 周计划只在用户选择该产物时生成。

范围控制：

- 单账号默认详读 TOP 5；用户要求深度版或数据足够时扩展到 TOP 10。
- 2-5 个账号先给比较矩阵，再写每个账号的独有结论；不重复相同的背景、方法和建议。
- 同一数据只出现一次；表格已表达的内容不再用长段落重复。

用户要可视化报告时，生成单页 HTML：

- 首屏展示头像、账号名、核心判断和关键数据。
- 热度榜用排名列表；内容主线用条形图/矩阵；选题用表格。
- 展示真实封面；统一放在 3:4 框内并使用 `object-fit: contain`，不裁切原图。
- 账号和笔记尽量保留可点击链接。
- 不显示“Skill 运行步骤”“输出结构建议”等内部说明。

输出使用当前工作区相对路径，不写创建者电脑的绝对路径。

## 隐私与合规

- 只分析公开可见内容或用户主动提供的数据；不绕过登录、验证码、权限或平台风控。
- 未授权时不安装软件、不读取 Cookie、不登录账号。
- 示例只使用虚构用户；不复用当前用户的公司、履历、姓名、账号、产品、城市或历史对话信息。
- 对搞钱、副业、收益类内容不写夸张承诺。

## 完成前检查

- 路由正确：有链接不追问；找对标才做定位问答。
- 找对标时第一轮已经询问是否有现成账号链接，没有延迟到第二轮。
- 模糊大类没有触发搜索；搜索前已经确认具体品类、账号类型和目标用户。
- 访问失败或被拒绝时，已经提供连接器和截图两条可继续执行的路径。
- 第一次访问失败后已经检查并询问宿主 App 的内置解决方案，没有过早进入兜底。
- 清楚标注访问状态；没有把外部搜索包装成小红书站内搜索。
- 所有数据有日期和来源；不混淆互动数、点赞、收藏和收藏率。
- 实际查看 TOP 内容和真实封面，不只根据简介下结论。
- 每张被分析的封面都有图像证据；没有看见图片时没有猜测视觉元素。
- 已从封面标题、内容结构、公开互动、关键词和主页一致性解释内容表现；没有加入不可见的后台或转化数据。
- 已把相似标题和封面按母题分组比较；没有把成功复用草率判定为“同质化”，也没有把重复本身写成平台官方流量机制。
- 只有在可见发布日期和样本跨度足够时才做前中后期分析，分期依据和转折证据清楚。
- 个性化建议与用户定位一致；未知定位时不假设。
- 链接可点击，图片完整，未获取字段已标注。
- 报告不暴露工具调用、内部提示或当前用户私人信息。

默认使用中文，语气直接、具体、有运营判断。
