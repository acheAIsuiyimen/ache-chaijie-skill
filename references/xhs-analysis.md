# 小红书分析框架

仅在开始正式账号分析时读取。目标是让结论基于真实封面和公开数据，不把运营经验冒充平台官方算法。

## 封面视觉审计

先实际打开封面原图或完整截图。缩略图过小、被裁切、加载失败时，不做视觉结论，改为获取完整图片。

对每张 TOP 封面记录：

1. **封面文字**：逐字识别主标题；记录行数、字号层级、关键词、数字和结果承诺。
2. **视觉主体**：人物、产品、界面截图、前后对比、成果物或场景；主体是否一眼可辨。
3. **构图动线**：标题、主体、补充信息的阅读顺序；是否拥挤、遮挡或留白失衡。
4. **颜色与对比**：主色、强调色、背景、文字可读性；手机缩略图下是否仍清楚。
5. **点击钩子**：身份、痛点、反差、数字、时效、稀缺、结果或好奇心。
6. **可信证据**：真实界面、过程截图、成品、数据、案例、人物身份；区分证据与装饰。
7. **模板一致性**：字体、人物位置、边框、标签和品牌元素是否形成系列识别。
8. **适配判断**：哪些元素适合用户复用，哪些依赖博主体量、颜值、资源或既有人设。

不要只说“颜色醒目、信息清晰”。必须指出具体文字、位置、视觉元素及其作用。

输出表格：封面缩略图、笔记链接、可见互动、主标题、视觉结构、点击钩子、可信证据、可复制点、主要问题。

完成逐张审计后，再归纳：

- 最高频封面结构与标题句式。
- 高热样本与低效样本的视觉差异。
- 一套可复用封面公式，例如“结果数字 + 成果截图 + 适用人群”。
- 1-2 个适合用户定位的模板方向，不机械复制颜色或排版。

## 账号演化分期

只有能看到足够笔记发布日期和内容样本时才分析。优先覆盖至少 20 篇、横跨 3 个月；不足时缩小结论范围，无法形成阶段证据时直接省略本模块。

不要机械按数量平均切成三份。根据实际变化点划分：

- **早期探索**：最早可见样本中尝试了哪些主题、形式、标题和封面。
- **转折阶段**：从哪一批笔记开始出现稳定主题、形式变化或公开互动抬升；列出对应笔记与日期。
- **近期稳定**：最近可见样本保留了哪些栏目、视觉模板和内容承诺，又出现了什么新变化。

每个阶段只比较可见字段：

1. 一句话定位和主要受众表达。
2. 高频主题、栏目及内容形式。
3. 标题句式、封面结构和是否露脸。
4. 可见发布时间与更新间隔；页面样本不完整时不写发布频率。
5. 公开互动的相对变化；不把不同发布时间的绝对数字直接等同为内容优劣。
6. 阶段代表笔记、链接和可见证据。

每个阶段必须显示“起止时间 / 代表笔记及日期 / 当时做法 / 变化原因 / 关键保留”。最终总结“保留了什么、放弃了什么、哪个转折最关键、用户可复制哪一步”。不得推断团队扩张、投流、平台扶持、收入或内部策略。

如果只有少量日期样本，不要伪造连续时间线。改为“可见节点”，并标注样本空白期。

## 内容诊断

用 5 项诊断快速回答“现在怎样、为什么、怎么改”：

1. **定位与人群**：账号是否给了明确人群、场景和长期关注理由。
2. **选题与需求**：高频选题是解决购买决策、情绪共鸣、实用教程还是陪伴更新；哪类已有多篇数据验证。
3. **标题与封面**：高热与低效样本的关键差别；是否存在可反复使用的爆款拍摄场景、标题句式或封面结构。
4. **内容价值**：内容是否提供搭配细节、选择理由、款式信息、步骤或对比；是否只有好看但缺少可带走信息。
5. **主页承接**：昵称、简介、置顶和近期内容是否对同一承诺；新访客能否立即知道为什么关注。

每项最多 4 行：现状、证据、判断、动作。没有证据时省略，不用空话填满。

## 小红书内容机制

不要声称掌握平台内部权重。只分析当前能看见的内容设计与公开信号：

1. **封面标题设计**：是否在缩略图环境中明确回答“给谁、解决什么、得到什么”。只评价设计，不推测点击率。
2. **内容结构**：可见首屏、正文或视频内容是否兑现承诺，是否易扫读，教程是否展示过程和结果物。
3. **可收藏结构**：是否包含清单、模板、步骤、参数、工具或完整案例。只判断结构，不推断真实收藏原因。
4. **公开评论需求**：只根据实际读到的评论归纳提问、经历、分歧、求资源或行动反馈；未读取评论时省略。
5. **关键词承接**：标题、正文、话题和账号定位中是否出现具体搜索词。不得推测搜索排名、搜索流量或收录结果。
6. **主页一致性**：头像、名称、简介、置顶和可见内容系列是否保持同一承诺。不得推测主页访问或关注转化。
7. **持续关注理由**：从可见栏目、连续实验、专业身份、资源或人格表达判断账号提供了什么长期价值；不得写实际涨粉效果。

区分内容形态：

- 图文教程：重点看首图点击、步骤完整性、截图证明和收藏价值。
- 视频：只有实际播放或取得完整内容时，才看开头、信息节奏、演示过程和字幕；只看到封面时不分析视频内容。
- 测评/种草：重点看使用场景、对比标准、真实体验和购买决策信息。
- 直播切片：重点看冲突点、利益点、上下文完整度和直播承接。

## 爆款资产复用

不要把相似标题、重复封面或反复选题直接判定为创意不足。先回溯：这个拍摄场景、封面结构或标题句式，是否曾经出现过明显高热样本。若是，将它定义为“爆款资产”，再分析后续复用。

将相似笔记按“同一人群 + 同一需求 + 同一结果承诺 + 同一表达结构”分组，每组尽量取 3 篇以上，比较发布日期、可见互动、具体案例和视觉差异。

区分三种重复：

1. **有效复用**：需求和结构不变，更换具体场景、商品、季节、案例或证据；多篇仍有相对较高的公开互动。
2. **结构迭代**：保留识别度，持续调整关键词、主体位置、标题长度或内容证据；观察哪个变体更稳定。
3. **机械复制**：标题、封面和内容几乎不变，没有新场景或新信息；多次发布的公开互动持续走低。

输出“爆款资产复用”表：资产名称、首个高热样本及日期、后续笔记链接、重复元素、变体元素、公开互动范围、判定、可复制规则。

优先给出这个结论：“爆款可以重复，重复后数据会波动。该账号复用的是已验证的识别度和用户需求，不是保证每篇爆。”然后用数据证明。

可以将“经验证的需求和内容结构能够被重复利用”写为运营方法；不得声称“平台会奖励重复内容”。后者需要平台官方证据，公开笔记数据只能支持账号层面的合理推断。

## 对标学习与账号发展

不要把“学习博主”写成标题清单。先输出一句迁移公式：`固定人群与需求 × 可持续拍摄场景 × 可重复内容结构 × 每篇新信息`。

再给出三阶段发展路径，篇数根据用户产能调整：

1. **起号基建**：明确一类核心人群、一个高频场景、一个可持续拍摄地点/表达方式、2-3 个内容支柱和主页承诺。说明初期必须保持不变的元素。
2. **验证与固化**：连续测试同一结构，每次只改一个变量。当同一结构至少 2-3 次高于账号常态数据，或反复出现同类评论需求，将它固化为账号资产。
3. **放大与扩展**：保留核心人群和爆款资产，用合集、对比、教程或相邻场景提高信息价值。只有核心内容能持续命中时才扩品类或扩人群。

每阶段只输出：目标、必做任务、保持不变、进阶信号。进阶信号优先使用相对信号，如“多篇高于自身中位水平”和“评论反复出现同类需求”，不为所有账号硬设统一点赞门槛。

最后输出“向该博主学什么 / 不学什么”：学需求、生产系统、爆款资产和发展顺序；不直接搬拍摄地点、原标题、照片和依赖博主体量的做法。

## 证据等级

每个关键结论只使用：

- **可见事实**：页面、图片或公开数字能够直接证明。
- **合理推断**：多个样本和行为逻辑支持，但不是平台官方结论。

以下字段除非用户提供后台数据，否则不出现在分析和报告中：曝光、点击率、阅读时长、完播率、流量来源、搜索流量、主页访问、单篇涨粉、成交和转化率。

不要仅凭点赞高就称为“高转化”，不要仅凭收藏高就计算收藏率，也不要用单篇内容概括整个账号。某一模块没有可见证据时直接省略，不用“无法确认”占据报告篇幅。
